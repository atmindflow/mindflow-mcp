Nested Subagents

Unlock infinite agent hierarchies with configurable depth control

What are Nested Subagents?

OpenClaw now supports sub-sub-agents - allowing agents to spawn their own subagents, which can spawn their own subagents, and so on. This creates a powerful hierarchy for complex task decomposition and parallel execution.

Why it Matters

Break down complex workflows into manageable layers. Each agent can delegate specialized tasks to child agents, creating a self-organizing system that scales with your problem's complexity while maintaining full control over depth and breadth.

How it Works

Configure maximum spawn depth and children per agent. Set maxSpawnDepth: 2 to allow agents to spawn subagents, which can spawn their own subagents (2 levels deep). Control resource usage with maxChildrenPerAgent.

Configuration

Add this to your OpenClaw configuration file:

agents:
  defaults:
    subagents:
      maxSpawnDepth: 2
      maxChildrenPerAgent: 5
