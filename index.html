OpenClaw 2026.2.17

Real‑time messaging UX, deterministic orchestration, and iOS quality‑of‑life — plus safer automation defaults.

Read the release Get DMG Get ZIP
Published: 2026‑02‑18 Latest

What’s new (highlights)

  • Slack native single‑message streaming: stream partial output into one message with chat.startStream/appendStream/stopStream, with automatic fallback.
  • /subagents spawn: deterministic subagent activation from chat commands — predictable, routeable orchestration.
  • iOS Share Extension: send URL/text/image to the gateway via agent.request (delivery‑route fallback + optional receipt acks).
  • Anthropic upgrades: opt‑in 1M context (params.context1m: true) and anthropic/claude-sonnet-4-6 support with forward‑compat fallback.
  • Cron safety & control: split webhook delivery from announce; validate HTTP(S) targets; new deterministic stagger controls (--stagger, --exact).
  • Telegram UX polish: inline button style (primary/success/danger) + surfaced reactions as system events.

Deep dives

Slack streaming that feels native

Cleaner threads, fewer pings, and better readability during long answers by streaming edits into a single message.

chat.startStream → chat.appendStream → chat.stopStream
Defaults: streaming on, graceful fallback on failure

Deterministic subagent orchestration

/subagents spawn gives you a reliable switch to activate a specific helper from chat, with clear behavior for one‑off calls and contextual task‑message prefixes.

/subagents spawn
Notes:
• one‑off calls disable polling subagents
• spawned task messages are context‑prefixed

iOS Share → instant agent workflows

Share to OpenClaw, route into the gateway, and trigger actions like "summarize link", "extract todos", or "draft reply" — fast on‑device loops.

Share Sheet → OpenClaw (iOS) → gateway agent.request
