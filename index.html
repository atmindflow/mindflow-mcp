Changes

New capabilities

  • Discord: Components v2 (buttons, selects, modals, attachment‑backed file blocks).
  • Discord: CV2 containers, embeds passthrough, exec approval UX refinements.
  • Plugins: llm_input & llm_output hook payloads for extensions (#16724).
  • Sub‑agents: nested sub‑agents with configurable depth + limits (#14447).
  • Slack/Discord/Telegram: per‑channel ack reaction overrides (#17092).
  • Cron/Gateway: finished‑run webhook notify + dedicated cron.webhookToken.
  • Channels: unified probe/token base types across core + extensions.
Fixes

Reliability & security

  • Security: SHA‑256 sandbox config hashing; block dangerous Docker configs.
  • Security/Logging: redact Telegram tokens; sanitize inbound chat; stricter role matching.
  • Sandbox: stable hashing; safer installers; cap web fetch size pre‑parsing.
  • Gateway/UI: better scope handling; safer status; stricter agent keys.
  • Memory/FTS: Unicode‑aware tokenization; correct time‑aware memory files.
  • Telegram/Discord: robust retries; de‑dup replies; stable streaming.
  • TUI/CLI: ANSI‑aware filtering, stable wrapping, better paste handling.
How it works

Upgrade paths and configuration

  • Discord Components v2 — No code changes required for basic actions. For modals/file blocks in skills, register component handlers and pass file attachments via CV2 containers.
  • Sub‑agents — Opt‑in depth via agents.defaults.subagents.maxSpawnDepth (e.g. 2) and limit fan‑out with maxChildrenPerAgent (default 5).
  • Plugin LLM hooks — Implement llm_input/llm_output in extensions to observe prompts and completion metadata for analytics, red‑teaming, or safety checks.
  • Security upgrades — Config hashing moves to SHA‑256, sandbox policy blocks risky Docker flags, secrets are redacted from logs by default.
