WebGL background (technical notes)

The background is a tiny, dependency-free WebGL1 program. It renders a full-screen quad and drives a fragment shader with time + resolution uniforms. Animation is automatically disabled for users with prefers-reduced-motion.

1) Context creation

We request a WebGL1 context with alpha enabled. Alpha allows the shader output to be blended subtly behind the content layer.

2) Geometry

A single buffer contains 6 vertices (two triangles) covering clip space (-1..1). No per-frame buffer uploads; the GPU reuses the static geometry.

3) Shader pipeline

The vertex shader forwards positions directly to gl_Position. The fragment shader computes a smooth gradient by combining sin/cos waves, then outputs a low-alpha color for a ‚Äúglow‚Äù effect.

4) Uniforms

We update u_time on every frame (seconds since start) and u_resolution on resize. This keeps animation consistent and aspect-correct.

5) Animation loop

Rendering runs via requestAnimationFrame. The loop pauses when the document is hidden and is disabled entirely when reduced motion is preferred.

6) DPR handling

The canvas backing store uses devicePixelRatio to remain sharp on Retina displays. We set the WebGL viewport to match the physical pixel dimensions.
