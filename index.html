Superpowers v4.1.1 for Mindflow MCP

This release turns the Github MCP server into a first-class orchestrated agent: safer multi-tool flows, project-aware defaults, and a landing page wired to https://atmindflow.github.io/mindflow-mcp/ so you can point agents and humans to one canonical entry.

Highlights

core changes Safer, more ergonomic GitHub MCP orchestration
Designed for "MCP-Orchestrating Agent" system prompts.
  • 1 Project landing is now MCP-native. The root index.html is managed as a single, orchestration-friendly artifact, making it trivial for your tools to SWcreate_or_update_file on main without worrying about assets or build steps.
  • 2 GitHub Pages wiring clarified. The page is intentionally placed at the repository root so that https://atmindflow.github.io/mindflow-mcp/ resolves cleanly for both humans and agents, no additional build system required.
  • 3 Release-notes layout optimized for LLMs. Content is structured with semantic sections (<section>, <article>, labeled lists) to make it easy for tools to summarize, diff, or generate follow-up tasks from the page itself.
Why this matters: for MCP-aware agents, the landing page doubles as both documentation and a stable target for scripted updates in CI-like flows.

Details

ux & safety Crafted for orchestrated GitHub flows
  • A Zero external assets. All typography, layout, and chrome are done with inline CSS and zero JS, keeping the page fully static, deterministic, and extremely cheap to serve.
  • B Single-file contract. Overwriting index.html is safe for agents: no bundler, no build pipeline, no extra manifests. This is ideal for MCP tools that only know how to push or update discrete files.
  • C Readable by design. Headings, lists, and code samples are formatted to be easy both on the eye and for downstream parsing (summaries, changelog extraction, or release automation).
M How a typical MCP orchestrator would reproduce this change

Superpowers v4.1.1 is designed so a tool-using agent can redo this deployment in 3 JSON-RPC steps:

  1. Discover create_or_update_file via tools/list on the GitHub MCP server.
  2. Select the index.html path and confirm the main branch as the deployment target.
  3. Call create_or_update_file with the generated HTML payload and a clear commit message, then surface the resulting commit URL or live Pages URL.
