Purpose of this page

This is a dedicated release note page for @refinedev/supabase@6.0.2. It intentionally highlights only the latest patch behavior change: how refine’s liveProvider builds Supabase Realtime postgres_changes filters.

What changed (single patch)

  • Supabase accepts one filter string per postgres_changes subscription.
  • When multiple filters are provided, refine now uses the first valid filter.
  • It also logs a warning instead of silently producing an invalid payload.
