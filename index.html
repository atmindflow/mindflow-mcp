openclaw 2026.2.17

The latest stable release upgrades real-time chat UX (native Slack streaming + Telegram button styling), makes orchestration more predictable (/subagents spawn), expands Anthropic model support (opt‑in 1M context + Sonnet 4.6), and unlocks faster mobile workflows (iOS Share Extension + Talk improvements).

What’s new (release highlights)

  • Native Slack single-message streaming: OpenClaw can stream into one Slack message using chat.startStream/appendStream/stopStream, keeping replies readable and aligned with threading rules.
  • iOS Share Extension: share a URL, text snippet, or image to OpenClaw and it’s forwarded directly to the Gateway via agent.request with delivery-route fallbacks.
  • Anthropic 1M context (opt-in): enable the beta header via params.context1m: true for Opus/Sonnet when you need extremely long context windows.
  • Anthropic Sonnet 4.6: adds anthropic/claude-sonnet-4-6 with forward-compat fallback when catalogs lag behind.
  • Deterministic orchestration: new /subagents spawn command for predictable subagent activation from chat commands.
  • Telegram UX: inline buttons now support styles (primary/success/danger) and optional reaction notifications as system events.
  • Cron safety & control: per-job webhook delivery separation + better recurring schedule timing with default stagger, --stagger, and --exact overrides.

Source: official v2026.2.17 release notes.

Deep dive into the new features

1) Slack: native single-message streaming

Instead of sending a burst of partial messages, OpenClaw can stream progressively into a single Slack message. This improves readability, reduces notification noise, and keeps reply threading consistent with your configuration.

# Uses Slack native stream API
# - chat.startStream
# - chat.appendStream
# - chat.stopStream

# Behavior
# - enabled by default
# - falls back to non-stream delivery when streaming fails

Why it matters: real-time answers without turning channels into spam feeds.

2) iOS: Share Extension → Gateway agent.request

Share from Safari/Photos/any app straight into OpenClaw. The extension forwards URL/text/image payloads to the Gateway and supports delivery-route fallbacks plus optional receipt acknowledgements.

# Workflow
1) iOS Share
2) Choose "OpenClaw"
3) Content is forwarded to gateway agent.request

# Great for
- summarize a link
- extract tasks from a screenshot
- draft a reply

Why it matters: zero copy/paste friction and better attachment fidelity.

3) Anthropic: opt-in 1M context + Sonnet 4.6

v2026.2.17 adds an opt-in beta header for ~1M context on supported Anthropic models, and first-class support for Sonnet 4.6 across aliases/defaults with forward-compat fallback.

# Enable 1M context (opt-in)
params:
  context1m: true

# Select Sonnet 4.6
anthropic/claude-sonnet-4-6

Why it matters: long projects fit in-context, and model selection stays resilient during provider rollouts.

4) /subagents spawn: deterministic subagent activation

You can now trigger subagents from chat commands in a deterministic way. This is useful when you want a specific “helper agent” behavior reliably.

/subagents spawn

Notes
- task messages are prefixed with context
- polling subagents are disabled for one-off calls

Why it matters: repeatable workflows for triage, scheduling, and automation.

5) Cron: safer webhooks + predictable schedule timing

Cron now separates per-job webhook delivery from announce delivery, enforces valid HTTP(S) webhook URLs, and adds deterministic stagger controls for recurring schedules.

openclaw cron add --stagger <duration>
openclaw cron edit --stagger <duration>
openclaw cron add --exact

# delivery.mode = "webhook" (per job)

Why it matters: fewer surprises in production automations and clearer delivery semantics.

6) Telegram: styled inline buttons + reaction notifications

Inline buttons now support styles (primary, success, danger), and Telegram can surface user reactions as system events via configurable scopes.

# Button styles
primary | success | danger

# Reaction events
channels.telegram.reactionNotifications: <scope>

Why it matters: clearer interactive flows and better observability.
