openclaw 2026.2.17

A release that pushes streaming and multi-channel UX forward (Slack streaming, Telegram button styles), improves subagent orchestration (new /subagents spawn), expands model support (Anthropic 1M context beta + Sonnet 4.6), and adds practical mobile workflows (iOS Share Extension + Talk background listening).

What’s new in v2026.2.17

  • Anthropic: opt‑in 1M context beta header — Enable params.context1m: true for Opus/Sonnet to send anthropic-beta: context-1m-2025-08-07.
  • Models: Anthropic Sonnet 4.6 support — Adds anthropic/claude-sonnet-4-6 with forward-compat fallback when provider catalogs lag.
  • Commands/Subagents: /subagents spawn — Deterministic subagent activation directly from chat commands.
  • Slack: native single-message streaming — Uses chat.startStream/appendStream/stopStream, enabled by default with fallback when streaming fails.
  • Telegram: inline button style + reaction events — Button styles (primary|success|danger) and optional reaction notifications.
  • iOS Share Extension — Share URL/text/images to the gateway via agent.request, with route fallback and optional receipts.
  • iOS Talk: Background Listening + Voice Directive Hint — Keep Talk active in background (opt-in) and optionally disable token-heavy voice-switch hints.
  • Cron/Gateway: webhooks + scheduling control — Separate per-job webhook delivery, enforce valid URLs, and add --stagger/--exact.
  • Security & reliability — Includes credential-theft mitigation (OC-09) and stronger context guards for tool output.

How it works

1) Turn on 1M context (Anthropic) when you actually need it

v2026.2.17 adds an explicit opt-in switch that maps to Anthropic’s 1M context beta header—keeping default behavior conservative while letting you unlock huge context windows for deep research or long transcripts.

agent:
  model: anthropic/claude-opus-4-6
  params:
    context1m: true   # adds: anthropic-beta: context-1m-2025-08-07

Why it matters: big context only when needed = lower cost + less accidental context bloat.

2) Sonnet 4.6 support with forward-compat fallbacks

Adopt anthropic/claude-sonnet-4-6 immediately, even if upstream model catalogs temporarily lag. OpenClaw keeps aliases/defaults aligned and falls back cleanly.

openclaw agent --message "Summarize this thread" --model anthropic/claude-sonnet-4-6

Why it matters: faster upgrades without brittle configuration churn.

3) Deterministic orchestration: /subagents spawn

Spawn subagents intentionally (not implicitly) to keep workflows predictable—perfect for runbooks, multi-step ops, and planner/executor patterns.

/subagents spawn planner
/subagents spawn executor

Why it matters: clearer control over parallelism, cost, and responsibility boundaries.

4) Slack streaming that feels native

Slack now supports true single-message streaming via chat.startStream/appendStream/stopStream. Reply threading stays aligned, and the system falls back gracefully if streaming fails.

channels:
  slack:
    stream:
      enabled: true

Why it matters: faster perceived latency and fewer message fragments in busy channels.

5) Mobile-first capture: iOS Share → Gateway → Agent

Share a URL, text, or image from any app and the iOS Share Extension forwards it directly into OpenClaw via agent.request, with delivery-route fallback and optional receipts.

Share Sheet → OpenClaw → gateway agent.request → your chosen channel/session

Why it matters: capture “in the moment” context without copy/paste or switching apps.

6) Telegram buttons get intent: primary|success|danger

Inline buttons can now declare a style, and reactions can be surfaced as system events (configurable scope). This is a small change that makes assistants feel more native and predictable.

{
  "buttons": [
    { "text": "Approve", "style": "success", "action": "approve" },
    { "text": "Reject",  "style": "danger",  "action": "reject" }
  ]
}

Why it matters: clearer UI = fewer wrong taps in high-stakes flows.

7) Cron gets stricter webhooks + better scheduling

Per-job webhooks are now separated from announce delivery, URLs are validated as HTTP(S), and recurring jobs get deterministic staggering to reduce top-of-hour spikes.

openclaw cron add --stagger 30s
openclaw cron edit --exact

Why it matters: fewer collisions, safer automation, and more predictable integrations.
