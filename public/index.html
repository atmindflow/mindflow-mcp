OpenClaw Latest release v2026.2.15

What’s new in v2026.2.15

A faster, safer, more interactive OpenClaw. This release focuses on richer native interactions, safer defaults, and deeper automation hooks — while keeping your personal AI assistant always-on and in control across your channels.

Install & onboard

npm install -g openclaw@latest
openclaw onboard --install-daemon

Talk to your assistant

openclaw agent --message "Ship checklist" --thinking high

Highlights

  • Discord Components v2

    Buttons, selects, modals, and attachment-backed file blocks for rich, native Discord interactions.

  • Plugins: prompt I/O hooks

    New llm_input and llm_output hooks let extensions observe prompt context and model output usage.

  • Nested sub‑agents

    Enable sub‑sub‑agents with depth control and child limits for scalable, multi‑agent workflows.

  • Per‑channel acks

    Override ack reactions per Slack/Discord/Telegram account or channel for platform‑native UX.

  • Cron & webhooks

    Finished‑run notifications and dedicated tokens for secure outbound cron webhook delivery.

  • Security hardening

    SHA‑256 sandbox hashing, strict sandbox config validation, secret redaction, and safer defaults across the stack.

How it works (1‑minute tour)

OpenClaw routes your chats from WhatsApp/Telegram/Slack/Discord/etc. through a local Gateway control plane. The Gateway coordinates sessions, tools, and apps (macOS/iOS/Android), so your assistant can listen, speak, browse, and automate securely — all from your own devices.

Channels → Gateway (control plane) → Agents & Tools → Replies back to your channels
