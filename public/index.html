OpenClaw 2026.2.17

A release focused on real-time messaging UX, deterministic orchestration, and safer automation — plus meaningful iOS upgrades.

Read the release Get DMG Get ZIP
Published: 2026-02-18 Latest

Release highlights

  • Slack native single-message streaming: stream partial output into one Slack message using chat.startStream/appendStream/stopStream, with fallback to normal delivery.
  • /subagents spawn: deterministic subagent activation from chat commands — ideal for predictable workflows and routing.
  • iOS Share Extension: share URL/text/image into OpenClaw; content is forwarded to the gateway via agent.request with delivery-route fallback.
  • Anthropic upgrades: opt-in 1M context header via params.context1m: true + support for anthropic/claude-sonnet-4-6 with forward-compat fallback.
  • Cron safety & control: split per-job webhook delivery (delivery.mode="webhook") from announce delivery, validate webhook URLs, and add deterministic schedule stagger controls (--stagger, --exact).
  • Telegram UX: inline button style support (primary/success/danger) + reaction notifications surfaced as system events.

Everything on this page is sourced from the official v2026.2.17 release notes.

Why these changes matter

Slack streaming that feels native

Streaming into a single message is a big usability win: fewer notifications, cleaner threads, and better readability during long answers.

# Slack native stream API
chat.startStream → chat.appendStream → chat.stopStream

# Defaults
- streaming enabled
- auto fallback when streaming fails

Deterministic orchestration with subagents

/subagents spawn gives you a reliable switch to activate a specific helper subagent from chat commands.

/subagents spawn

Notes
- one-off calls: polling subagents disabled
- task messages are prefixed with context

Faster mobile workflows

The iOS Share Extension sends shared content directly to the gateway, enabling quick “summarize this link”, “extract tasks from screenshot”, and “draft a reply” flows.

iOS Share → OpenClaw → gateway agent.request
